STEP 1/3
Priority Triage — QuickSort
O(n log n) · Iterative · Randomised Pivot
Iterative QuickSort with a randomised pivot re-ranks all 10 orders by priority score (1–10 descending).
Tie-break: heavier orders come first so Knapsack packs the most impactful items early.
Result: guaranteed SLA-critical / high-urgency orders get the first available van slot.

STEP 2/3
Van Packing — 0/1 Knapsack DP
O(n × W) · W = 50 kg/van · 1D DP + Backtracking
Dynamic programming fills each 50 kg van to maximise total priority-value, not just weight.
Backtracking recovers the exact item selection from the DP table. Remaining orders go to the next van.
STEP 3/3
Route Optimization — Dijkstra + TSP
Dijkstra all-pairs + Branch & Bound (≤8) / 2-opt + Or-opt (≥9)
Dijkstra computes pairwise road distances with a traffic factor (Haversine × 1.4).
TSP Branch & Bound with MST pruning for ≤8 stops (exact). Nearest-neighbour seed + 2-opt + Or-opt for ≥9 (heuristic, ~15% of optimal).
NP-hard problem — the heuristic trades ~15% optimality for polynomial time.
